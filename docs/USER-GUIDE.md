## 1. What it is, what you get

AI Article Factory is a **Claude Code skill suite**. Give it a sentence or a pile of materials; it produces a publish-ready article plus its platform versions.

**Per article you get:**
- `article-final.md` — the illustrated body (for your website)
- `article-final.docx` — Word version (review / clients / offline; figures embedded)
- `article-linkedin.md` / `article-x.md` — LinkedIn / X copy (on request)
- Figures: data charts, flow/structure diagrams, a cover
- An evidence ledger — every fact has a traceable source

**Core idea: the AI runs the line, you guard the gates.** Automation is not unattended — quality depends on you checking a few key points.

| Required | Notes |

| A topic or some materials | a sentence, a product, or a folder of files — any one is enough |

**Optional tools** (works without them, you just lose one capability):

> A **plain Markdown draft needs zero dependencies**. Tools are for "want it better," not to get started.

## 3. Install (once)

**Install the skill** (pick one):
```bash
# A. Clone into your Claude Code skills folder
git clone https://github.com/weitzu-com/ai-article-factory ~/.claude/skills/ai-article-factory

# B. Via the skills CLI
npx -y skills@latest add weitzu-com/ai-article-factory

# C. Copy the whole folder into your project's .claude/skills/
```

**Install optional tools** (macOS):
```bash
brew install pandoc                            # Word export
npm install -g @mermaid-js/mermaid-cli         # flow/structure diagrams
python3 -m pip install matplotlib numpy        # data charts
```
> On a clean Mac you first need [Homebrew](https://brew.sh) (for pandoc) and [Node](https://nodejs.org) (for mmdc).

**Required on macOS** to read materials from your computer →
**System Settings → Privacy & Security → Full Disk Access → enable the app that runs Claude Code, then restart it.**
Otherwise you'll see `Operation not permitted` and the AI can't read your files.

**The two most common gotchas:**
1. **`cd` into your own project folder first**, or articles get written inside the skill's install directory and you can't find them.
2. **The cover image is generated by you** — the AI gives you a prompt; make the image with any tool and save it as `assets/img/cover.png` (a placeholder is used until then, so it never blocks).

## 4. Your first article in 5 minutes

`cd` into your project, start Claude Code, then **just say what you want** — the skill auto-triggers:

```
"Write an article about <product> for my website."
"Scan ~/Downloads/research/ and give me 3 topic ideas, then write the best one."
"Use the files in this folder to write a buyer's guide for procurement managers."
```

It runs the **default "water" path** full-auto, pausing at only **2 points** for you:

```
your sentence / a materials folder
  → AI auto-files + drafts a blueprint     ←[confirm ①] optional; silence = continue
  → AI writes the draft (every fact sourced)
  → AI self-checks + figures + export       ←[confirm ②] glance, approve or tweak
  → article.md + article.docx (ready to publish)
```

Defaults stay out of your way: the AI infers the topic profile (**no forms**), the cover uses a placeholder, and only the website version is produced (say so to add LinkedIn/X).

## 5. You do only three things: review · confirm · publish

| # | You do | What to look at |
|---|---|---|
| **Review** | the topic card / audit report / platform copy | Is the angle right? Does every figure have a source? Does it read like a human wrote it? |
| **Confirm** | nod at the gates | Especially: **adjudicate any "disputed/unverified" rows** in the evidence ledger; release only when QA says ship |
| **Publish** | paste to website / LinkedIn / X and hit publish | Publishing is usually a manual step |

> Everything else is automatic. Want it more automatic (auto-pick the first topic) or less (confirm every step)? Just say so.

## 6. What the AI does behind the scenes (11 stages)

You don't have to memorize this, but knowing the flow helps you review:

```
P0 profile → P1 topic → P2 evidence★ → P3 angle → P4 outline → P5 draft
→ P6 visuals → P7 SEO/GEO → P8 QA★ → P9 export MD+Word → P10 publish → P11 monitor
```
★ = **hard gate**: no writing until the evidence ledger is complete (P2); no finalizing until QA passes (P8). These two are the quality fuses.

Six worker skills do the labor: `article-research` (topic + evidence), `article-write` (drafting, anti-AI-slop), `article-visuals` (figures), `article-optimize` (SEO/GEO), `article-qa` (QA), `article-export` (Word).

## 7. Using your own materials

Materials come from two places, both usable:
1. **Any path you point to** — "use the files in `~/Downloads/projectX/`", or even "scan my computer for docs about X".
2. **A shared library** — keep reusable authoritative sources (standards PDFs, spec sheets, white papers); file once, reuse across articles.

For each item the AI: **discovers → extracts (PDF/Word/Excel/web/image) → registers it into the evidence ledger with a page/clause locator → grades it** (primary source ✅ / disputed / unverified; competitor blogs are never treated as authority).

> **A realistic expectation:** the more structured the material, the more fully automatic it is. With a large technical PDF, the AI may need you to point it at "this section / this clause" — that's AI-*assisted* filing, and it's normal.

## 8. Getting your outputs

Each article is one self-contained folder `articles/<date-slug>/`:
```
├── 00-profile … 06-audit-report     ← process artifacts (AI fills these; you rarely touch them)
├── assets/img/                       ← figures (incl. cover)
├── <slug>-final.md                   ← website body (illustrated)
├── <slug>-final.docx                 ← Word
├── <slug>-wp.md                      ← WordPress version
└── <slug>-linkedin.md / -x.md        ← social copy (on request)
```
Run figures / export by hand (the AI usually does this for you):
```bash
bash <skill>/scripts/render-figures.sh articles/<slug>/   # figures
bash <skill>/scripts/export.sh articles/<slug>/ <slug>     # Word
```

## 10. A full worked example: the API 15S run

A **real run** (non-metallic pipe industry) showing materials → finished article.

**Step 1 · Give materials + topic**
> "Use the API 15S PDF in `shared-materials/standards/` to write a standard explainer for procurement managers."

**Step 2 · AI auto-files + reads the standard**
The AI reads the real PDF, extracts real clauses, and logs them into `02-evidence-ledger.md`, e.g.:
- API 15S scope [§1, p.1]
- Three-layer build: liner + reinforcement + cover [§3.1.18/3.1.40/3.1.7]
- MPR derived from a 175,000-hour regression curve reference time [§3.1.39, p.7]
- Blister test: decompression rate ≥ 70 bar/min, inspected at 20× [§4.2.1.3, p.10]

Each carries a **clause number + page** — that's "evidence before sentences."

**Step 3 · AI drafts + makes figures**
- Answer-first, every fact tagged `[C#]` back to the ledger
- 3 figures: three-layer structure (Mermaid) + pressure hierarchy MPR≥NPR≥MOP (matplotlib) + cover

**Step 4 · You review**
Open `06-audit-report.md`: all `[C#]` reconcile to the ledger, 0 unverified → ship-eligible. Spot-check 2–3 clause citations.

**Step 5 · Get the outputs**
In `articles/2026-06-08-api-15s-…/`: `*-final.md` (illustrated), `*-final.docx` (Word, 3 figures embedded), `*-linkedin.md`.

**Step 6 · You confirm and publish**
After replacing the illustrative ratios in Figure 2 with real product values, publish to your site + LinkedIn (link in the first comment).

> **Takeaway:** the AI hands you a **sourced, traceable draft**; fact-checking and publishing are your call.

## 11. Tips for a good article

- **Evidence first**: a number with no source gets marked `unverified` and softened, never invented. Seeing `unverified` tells you to add a source or cut it.
- **Gates are your friend**: P2 (evidence) and P8 (QA) block half-baked work. Being blocked isn't a bug — it's the fuse doing its job.
- **First-hand experience is gold**: give it your real data, failures, and contrarian judgment — that's what neither AI nor competitors can copy, and what gets cited.
- **Stop when it says stop**: if the audit says "no-ship," send it back; don't force-publish.
- **Prefer a table to a picture of a table**: tables are extractable, accessible, and indexable.

## 12. Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `Operation not permitted` (can't read files) | macOS privacy not granted | System Settings → Privacy & Security → **Full Disk Access** → enable the host app, **restart** it |
| Can't find the article after a run | didn't cd into your project | `cd` to your working dir before running |
| Diagrams don't render | mermaid-cli missing | `npm i -g @mermaid-js/mermaid-cli`; missing → auto-skipped |
| Chinese/CJK text shows as boxes | font missing | bundled `figlib` registers CJK fonts; non-Mac may need to install one |
| No images in the Word file | path issue | use `export.sh` (handles image embedding) |
| Cover stays empty | cover is manual | make it per `cover-prompt.md`, save `assets/img/cover.png` |
| It didn't read a big PDF fully | large files need guidance | tell it "look at section X / clause Y" |

## 14. FAQ

**Q: I'm non-technical — can I use it?**
Yes. On the default path you **never touch the CLI or fill a form** — you say a sentence, then review/confirm/publish.

**Q: Will it fabricate data?**
No. The rule is "evidence before sentences" — a fact with no source is marked `unverified` and softened, never stated as fact. So every number you see should have a source.

**Q: Do I have to install the two external SEO skill packs?**
No. The suite **bundles 6 worker skills** and runs the core pipeline standalone. The external packs are an optional enhancement for deeper research/scoring.

**Q: Can it publish to my own website?**
It produces Markdown (website/WordPress) + Word. A full WordPress-on-SiteGround + Yoast publishing checklist is in `references/publishing.md`.

**Q: Does it support non-English articles?**
Yes. It follows the language you write in; CJK renders correctly in figures, and the anti-AI-slop rules adapt per language.

**Q: How do I know if an article is ready to publish?**
Check the ship / no-ship verdict in `06-audit-report.md`. "Ship" plus your spot-check of the evidence = good to publish.
