# AI Article Factory · 用户使用手册

> 一句话：把**任意素材 / 任意选题**，变成**图文并茂、SEO + GEO 优化、Markdown + Word 双格式 + 多平台文案**的文章。
> **你（人）只做三件事：审核、确认、发布。** 其余——建档、选题、找证据、写作、配图、优化、质检、导出——交给 AI。
>
> 本手册面向**使用者**（操作工厂的人）。给 AI 看的指令在 `SKILL.md`，给开发者看的在 `AGENTS.md`。

## 1. 这是什么，你能得到什么

AI Article Factory 是一个 **Claude Code 技能套件**。给它一句话或一堆素材，它产出一篇可直接发布的文章，并配齐多平台版本。

**你每篇会得到：**
- `文章-final.md` — 图文并茂的正文（官网用）
- `文章-final.docx` — Word 版（评审 / 客户 / 线下留档，图已嵌入）
- `文章-linkedin.md` / `文章-x.md` — LinkedIn / X 平台文案（按需）
- 配图：数据图、流程/结构图、封面
- 证据台账：每个数据都有出处，可追溯

**核心理念：AI 跑流程，你守门。** 自动化 ≠ 无人化——质量靠你在几个关键点把关。

## 2. 开始前你需要准备什么

| 必备 | 说明 |
|---|---|
| **Claude Code** | 桌面版 / 命令行版均可 |
| 一个选题或一份素材 | 一句话、一个产品、或一个资料文件夹，任选其一 |

**可选工具**（缺了也能用，只是少某个能力）：
- `pandoc` — 导出 Word（缺了只给 Markdown）
- `@mermaid-js/mermaid-cli` — 流程/结构图（缺了自动跳过这类图）
- `python3 + matplotlib` — 数据图（缺了用 Markdown 表格代替）

> **纯 Markdown 草稿零依赖**——什么都不装也能出文字稿。工具是"想要更好"才装。

## 3. 安装（一次性）

**装技能**（三选一）：
```bash
# A. 克隆到 Claude Code 技能目录
git clone https://github.com/weitzu-com/ai-article-factory ~/.claude/skills/ai-article-factory

# B. 用 skills CLI
npx -y skills@latest add weitzu-com/ai-article-factory

# C. 直接把整个文件夹拷进你项目的 .claude/skills/
```

**装可选工具**（macOS）：
```bash
brew install pandoc                            # Word 导出
npm install -g @mermaid-js/mermaid-cli         # 流程/结构图
python3 -m pip install matplotlib numpy        # 数据图
```
> 全新 Mac 需先装 [Homebrew](https://brew.sh)（给 pandoc）和 [Node](https://nodejs.org)（给 mmdc）。

**macOS 必做一步**：若要让它读你电脑里的素材 →
**系统设置 → 隐私与安全性 → 完全磁盘访问权限 → 勾选你启动 Claude Code 的那个 App，然后重启该 App。**
否则会报 `Operation not permitted`，AI 读不到你的文件。

**两个最常见的坑：**
1. **先 `cd` 到你自己的项目文件夹再用**，否则文章会生成进技能安装目录里，你找不到。
2. **封面图要你手动生成回填**——AI 给提示词，你用任意画图工具出图，存成 `assets/img/cover.png`（不放也不阻塞，会用占位图）。

## 4. 5 分钟跑出第一篇

进项目目录、启动 Claude Code，然后**直接说人话**——技能会自动触发：

```
"帮我写一篇关于 <你的产品> 的文章，发官网。"
"扫一下 ~/Downloads/调研资料/，给我 3 个选题，再把最好的写出来。"
"用这个文件夹里的资料，写一篇面向采购经理的选型指南。"
```

它会按"默认水路"全自动走，只在 **2 个点**停下来等你：

```
你的一句话 / 一个素材夹
  → AI 自动建档 + 出蓝图        ←【确认①】可跳过，不说话就继续
  → AI 写稿（每个数据带出处）
  → AI 自检 + 配图 + 导出        ←【确认②】瞄一眼，确认或让它改
  → 文章.md + 文章.docx（可发布）
```

默认行为（不打扰你）：AI 自己推断选题档案（**不用填表**）、封面用占位图、默认只出官网版（要 LinkedIn/X **说一声**就加）。

## 5. 你只做三件事：审核 · 确认 · 发布

| # | 你做什么 | 具体看什么 |
|---|---|---|
| **审核** | 看选题卡 / 审核报告 / 各平台文案 | 方向对不对？数据有没有出处？读着像不像人写的？ |
| **确认** | 在门禁点点头放行 | 尤其：**证据台账里"存疑/冲突"的项**要你裁决；质检报告 ship 才放行 |
| **发布** | 贴到官网 / LinkedIn / X，点发布 | 平台发布通常需要人工操作 |

> 其余全自动。想提高自动度（让它自动选第一个选题），或降低（每步都要你点头），直接说就行。

## 6. AI 在背后做什么（11 阶段速览）

你不必记，但理解流程能帮你更好地审核：

```
P0 产品档案 → P1 选题 → P2 证据台账★ → P3 主题角度 → P4 大纲 → P5 写作
→ P6 配图 → P7 SEO/GEO优化 → P8 质检★ → P9 导出MD+Word → P10 发布 → P11 监测
```
★ = **硬门禁**：P2 没有齐全证据不许写作；P8 质检不过不许定稿。这两道是质量保险丝。

六个干活的子技能：`article-research`（选题+证据）、`article-write`（写作·反AI腔）、`article-visuals`（配图）、`article-optimize`（SEO/GEO）、`article-qa`（质检）、`article-export`（导出 Word）。

## 7. 用你自己的素材

素材有两个来源，AI 都能用：
1. **你指给它的任意位置** —— "用 `~/Downloads/项目X/` 里的资料"，甚至"扫一下我电脑里关于 X 的文档"。
2. **共享素材库** —— 放反复要用的权威资料（标准 PDF、规格书、白皮书），一次入库多篇复用。

AI 对每份素材的标准动作：**发现 → 提取（PDF/Word/Excel/网页/图片）→ 登记进证据台账（带页码/条款定位）→ 分级**（一手来源✅ / 存疑 / 冲突；竞品博客不作权威）。

> **真实预期**：素材越结构化，自动化越彻底；遇到大部头技术 PDF，AI 需要你帮忙指一下"看哪几节/哪个条款"——这是"AI 辅助建档"，正常。

## 8. 拿到你的产物

每篇文章是一个自包含文件夹 `articles/<日期-slug>/`：
```
├── 00-产品档案 … 06-审核报告      ← 过程产物（AI 自动填，你一般不用动）
├── assets/img/                    ← 配图（含封面）
├── <slug>-final.md                ← 官网正文（图文）
├── <slug>-final.docx              ← Word
├── <slug>-wp.md                   ← WordPress 版
└── <slug>-linkedin.md / -x.md     ← 社媒文案（按需）
```
手动出图 / 导出（一般 AI 已帮你跑，需要时可自己跑）：
```bash
bash <技能>/scripts/render-figures.sh articles/<slug>/   # 配图
bash <技能>/scripts/export.sh articles/<slug>/ <slug>     # 出 Word
```

## 10. 完整实例：用 API 15S 标准跑一篇

这是一次**真实跑过**的例子（非金属管道行业），演示"素材 → 成品"全过程。

**第 1 步 · 给素材和选题**
> "用 `02_共享素材/API标准/` 里的 API 15S PDF，写一篇面向采购经理的标准解读。"

**第 2 步 · AI 自动建档 + 读标准**
AI 读真实标准 PDF，抽出真实条款，登记进 `02-证据台账.md`，例如：
- API 15S 覆盖范围 [§1 Scope, p.1]
- 三层结构：内衬 + 增强层 + 外护套 [§3.1.18/3.1.40/3.1.7]
- MPR 基于 175,000 小时回归曲线参考时间 [§3.1.39, p.7]
- 起泡测试：降压速率 ≥70 bar/min，20× 放大检查 [§4.2.1.3, p.10]

每条都带**条款号 + 页码**——这就是"先有证据，再有句子"。

**第 3 步 · AI 写稿 + 配图**
- 答案前置、每个数据带 `[C#]` 回指台账
- 自动出 3 张图：三层结构图（Mermaid）+ 压力等级 MPR≥NPR≥MOP（matplotlib）+ 封面

**第 4 步 · 你审核**
看 `06-审核报告.md`：8 个 `[C#]` 全部对上台账、0 待核 → ship-eligible。你抽查 2-3 条条款引用是否准确。

**第 5 步 · 拿成品**
`articles/2026-06-08-api-15s-spoolable-pipe-explained/` 下：`*-final.md`（图文）、`*-final.docx`（Word，3 图嵌入）、`*-linkedin.md`。

**第 6 步 · 你确认并发布**
确认图 2 的压力比例需换成真实产品数值后，发 cnps.com + LinkedIn（文案链接放评论区）。

> **关键收获**：AI 给出的是**有出处、可追溯**的草稿；事实核对与发布，最终由你拍板。

## 11. 写出好文章的几个要点

- **证据优先**：没有出处的数据，AI 会标 `待核` 并软化措辞，不会硬编。你看到 `待核` 就知道该补来源或删。
- **门禁是朋友**：P2（证据）和 P8（质检）会拦住半成品。被拦不是 bug，是质量保险丝在工作。
- **一手经验最值钱**：把你的真实数据、踩坑、独到判断告诉它——这是 AI 和竞品都抄不走、也是被搜索/AI 引用的关键。
- **该叫停就叫停**：审核报告说"no-ship"时，让它回去改，别强发。
- **图能用表格表达就用表格**：表格 AI 能抽取、可访问、可索引，胜过把数据做成图。

## 12. 故障排查

| 现象 | 原因 | 处理 |
|---|---|---|
| `Operation not permitted`（读不到文件） | macOS 隐私权限未授权 | 系统设置→隐私与安全性→**完全磁盘访问**→勾选宿主 App 并**重启** |
| 文章生成后找不到 | 没先 cd 到自己项目 | 先 `cd` 到你的工作目录再用 |
| 流程图不出 | 缺 mermaid-cli | `npm i -g @mermaid-js/mermaid-cli`；缺了会自动跳过 |
| 图里中文变方框 | 缺中文字体 | 用内置 `figlib`（已注册中文字体）；非 Mac 需另装 |
| Word 里没有图 | 路径问题 | 用 `export.sh`（已自动处理图片嵌入） |
| 封面一直是空的 | 封面要手动生成 | 按 `cover-prompt.md` 出图，存 `assets/img/cover.png` |
| 读大 PDF 它没全读 | 大文件需引导 | 告诉它"看第几节/哪个条款" |

## 14. 常见问题 FAQ

**Q：我完全不懂技术，能用吗？**
能。默认水路下你**不碰命令行、不填表**——只说一句话，然后审核/确认/发布。

**Q：它会编造数据吗？**
不会。规则是"先有证据再有句子"——找不到出处的数据会被标 `待核` 并软化，不会写成确定事实。所以你看到的数字都该有来源。

**Q：必须装那两个外部 SEO 技能包吗？**
不必。本套件**自带 6 个 worker 技能**，核心流程独立可跑。外部包是"想要更深研究/打分"时的可选增强。

**Q：能发到我自己的网站吗？**
能。产出 Markdown（官网/WordPress）+ Word。WordPress on SiteGround + Yoast 的完整发布步骤见仓库 `references/publishing.md`。

**Q：支持中文文章吗？**
支持。跟随你说话的语言；配图中文不变方框。反 AI 腔规则也会按对应语言适配。

**Q：我怎么知道这篇能不能发？**
看 `06-审核报告.md` 的 ship / no-ship 判定。ship 且你抽查证据无误 = 可发。
